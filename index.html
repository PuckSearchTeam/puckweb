Puck is an architecture refactoring tool designed to enhance your development experience by providing bad dependencies identification within IntelliJ.

Weland is a language interpreted by Puck to design dependency rules by expressing coupling constraints. It allows to defind sets of Java code named elements such as package, class, attribute, method/constructor. Its core predicate is of the form "hide X from Y", where X and Y are sets. A dependency rule consists in the declaration of these sets followed by such a predicate and states that no element in Y must not depend on any element of X. More precisely, no element of the transitive closure of X can appear in the scope of any element of the transitive closure of Y.

Puck dynamically generates and highlights within the code all violations of Weland rules.
