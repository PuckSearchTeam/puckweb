Weland is a language interpreted by Puck to express coupling constraints rules. It provides the definition of sets containing of Java code entities such as package, class, attribute, method/constructor. Its core predicate is of the form "hide X from Y", where X and Y are sets. This rule expresses that no element in Y must not depend on any element of X. In other words, no element of X can appear in the scope of any element of Y

Puck is an architecture refactoring tool designed to enhance your development experience by providing bad dependencies identification within IntelliJ. It dynamically generates and highlights within the code all violations of Weland dependency rules.
